# AI Engineering Guide

AI를 활용한 작업 흐름을 한눈에 정리한 인터랙티브 가이드입니다.

이 문서는 거창한 방법론을 증명하거나 정답을 제시하기 위한 자료가 아닙니다.  
AI에게 일을 맡길 때 어떤 방식으로 요청하고, 어떤 자료를 함께 주고, 어떤 도구와 연결하면 작업 흐름이 어떻게 달라지는지 직접 파악하기 위한 가이드입니다.

예제 흐름은 **일정관리 앱 만들기**를 중심으로 구성되어 있습니다.  
단순히 “앱 만들어줘”라고 요청하는 방식에서 시작해, 산출물 작성, 도구 확장, 에이전트 구조까지 점점 넓혀가며 AI 활용 방식을 비교할 수 있도록 구성했습니다.

## 구성

이 가이드는 총 5개 챕터로 구성되어 있습니다.

### 00. Overview

LLM, Workflow, AI Agent, Orchestration의 차이를 전체 흐름으로 정리합니다.

핵심 질문은 간단합니다.

> 다음에 무엇을 할지 누가 결정하는가?

단순한 모델 응답에서 시작해, 정해진 흐름, 목표 기반 실행, 여러 실행 단위의 조율까지 자연스럽게 이어서 볼 수 있습니다.

### 01. Prompt Engineering

AI에게 원하는 결과를 얻기 위해 요청을 어떻게 구성할지 다룹니다.

일정관리 앱을 바이브코딩으로 만들어보는 흐름을 기준으로, 다음 내용을 확인합니다.

- 미션과 필수 기능 정리하기
    
- AI에게 역할 부여하기
    
- Few-shot 예시 제공하기
    
- 구현 기준 정리하기
    
- 결과물 수정 요청하기
    
- 작업물 리뷰 작성하기
    

목표는 프롬프트를 한 번에 완벽하게 쓰는 것이 아니라, 결과를 보며 요청을 점점 구체화하는 감각을 익히는 것입니다.

### 02. Context Engineering

AI에게 단순 요청만 던지는 것이 아니라, 작업에 필요한 자료와 기준을 함께 제공하는 방식을 다룹니다.

챕터 1에서 진행한 일정관리 앱을 기준으로, PRD, IA, 화면설계서, 기능설계서, 테스트 조건 같은 산출물을 만들어보고 이를 바탕으로 구현 흐름을 다시 잡아봅니다.

주요 체크 항목은 다음과 같습니다.

- SDD 산출물 작성하기
    
- 다른 LLM을 활용해 산출물 다듬기
    
- Claude 또는 Codex CLI의 plan 모드 사용해보기
    
- 스펙 수정 후 코드 반영하기
    
- TDD 산출물 작성하기
    
- 작업물 리뷰하기
    

핵심은 “요청 문장”만 바꾸는 것이 아니라, AI가 참고할 수 있는 맥락 자체를 정리하는 것입니다.

### 03. Extension

AI가 더 많은 작업을 할 수 있도록 외부 도구나 작업 방식을 연결하는 개념을 다룹니다.

주요 주제는 다음과 같습니다.

- MCP
    
- Skills
    
- Plugin
    
- MCP, Skills, Plugin의 차이
    
- 적용 전 확인할 체크리스트
    

이 챕터는 실습보다는 확인용에 가깝습니다.  
내 프로젝트에서 어떤 연결이 필요한지, 어떤 작업을 반복해서 쓰는지, 어떤 부분을 팀 단위로 정리할 수 있을지 점검하는 용도입니다.

### 04. Advanced

AI Agent를 더 넓은 구조로 이해하기 위한 내용을 다룹니다.

주요 주제는 다음과 같습니다.

- Harness Engineering
    
- Agent Orchestration
    
- Multi-Agent Orchestration
    

AI Agent를 단순히 모델 하나로 보지 않고, 모델을 둘러싼 실행 루프, 도구 호출, 컨텍스트 관리, 상태 관리, 실패 처리, 권한 관리까지 함께 보는 관점으로 정리했습니다.

## 주요 특징

- 단일 HTML 파일로 구성
    
- 좌측 대메뉴와 소메뉴 기반 탐색
    
- 챕터별 내용 전환
    
- 체크리스트 인터랙션
    
- 반응형 화면 지원
    
- 별도 빌드 과정 없이 브라우저에서 바로 확인 가능
    

## 실행 방법

저장소를 내려받은 뒤 HTML 파일을 브라우저에서 열면 됩니다.

```bash
git clone <repository-url>
cd <repository-name>
```

이후 `vibecoding guidance.html` 파일을 브라우저에서 실행합니다.

별도의 서버나 패키지 설치는 필요하지 않습니다.    

## 예제 미션

가이드에서 반복적으로 사용하는 예제 미션은 다음과 같습니다.

> 일정관리 앱을 바이브코딩으로 만든다.  
> 필수 기능은 할 일 추가, 완료 처리, 삭제, 날짜 정렬이다.

이 미션을 기준으로 챕터 1에서는 프롬프트를 다듬고, 챕터 2에서는 산출물을 만들어 구현 기준을 정리하고, 이후 챕터에서는 도구 연결과 에이전트 구조를 확장해서 바라봅니다.
